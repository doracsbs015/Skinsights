## рџљЂ Setup Guide

### Prerequisites
- Node.js 18+
- MongoDB (local or [MongoDB Atlas](https://cloud.mongodb.com))
- Google Gemini API key (optional but recommended)

---

### 1. Clone & Install

```bash
# Backend
cd skinsights/backend
npm install

# Frontend
cd ../frontend
npm install
```

### 2. Configure Environment Variables

```bash
cd backend
cp .env.example .env
```

Edit `.env`:
```
PORT=5000
MONGO_URI=mongodb://localhost:27017/skinsights
JWT_SECRET=your_super_secret_key_here_make_it_long
GEMINI_API_KEY=your_gemini_api_key_here
ADMIN_EMAIL=admin@yourdomain.com
```

> **Note:** The email you set as `ADMIN_EMAIL` will automatically receive admin privileges when they sign up.

### 3. Get a Gemini API Key (Optional)

1. Go to [Google AI Studio](https://aistudio.google.com)
2. Create a new API key
3. Paste it into `.env` as `GEMINI_API_KEY`

Without a Gemini key, the app falls back to pre-written explanations вЂ” everything else works normally.

### 4. Run the App

**Terminal 1 вЂ” Backend:**
```bash
cd backend
npm run dev
# Server starts at http://localhost:5000
```

**Terminal 2 вЂ” Frontend:**
```bash
cd frontend
npm run dev
# App opens at http://localhost:5173
```

## рџ§¬ How the Ingredient Analyzer Works

1. **Parse** вЂ” Ingredient text is cleaned (lowercase, stripped of punctuation) and split by comma
2. **Match** вЂ” Each ingredient is checked against the local 80+ ingredient database (exact then partial matching)
3. **Classify** вЂ” Ingredients are labeled as beneficial, risky, or unknown
4. **Skin-type Risk Check** вЂ” Cross-references your skin type's risk list for extra sensitivity
5. **Verdict** вЂ” `0 risky в†’ Suitable`, `1-2 risky в†’ Use with caution`, `3+ risky в†’ Not suitable`
6. **AI Explanation** вЂ” Result is sent to Google Gemini Pro for a personalized, readable explanation
